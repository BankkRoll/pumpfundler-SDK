# PumpFundler SDK

The PumpFundler SDK is a powerful TypeScript library designed to interact with the PumpFun protocol on the Solana blockchain. It provides a comprehensive set of tools for creating tokens, managing bonding curves, and executing trades.

## Importing the SDK

To use the PumpFundler SDK in your project, import it as follows:

```typescript
import { PumpFundlerSDK, PumpFundlerConfig } from "pumpfundler-sdk";
```

## Configuration

Before initializing the SDK, you need to create a configuration object:

```typescript
import { Connection, Commitment } from "@solana/web3.js";

const config: PumpFundlerConfig = {
  connection: new Connection("https://api.mainnet-beta.solana.com"),
  jitoFee: 1000000, // 0.001 SOL
  commitmentLevel: "confirmed" as Commitment,
  blockEngineUrl: "https://your-jito-block-engine-url.com",
  jitoAuthKeypair: "your-base58-encoded-jito-auth-keypair",
};
```

## Initializing the SDK

To create an instance of the PumpFundlerSDK, you need to provide a provider and the configuration:

```typescript
import { AnchorProvider, Wallet } from "@coral-xyz/anchor";
import { Keypair } from "@solana/web3.js";

const wallet = new Wallet(Keypair.generate());
const provider = new AnchorProvider(config.connection, wallet, {});
const sdk = new PumpFundlerSDK(provider, config);
```

## Core Functions

### Creating and Buying Tokens

The `createAndBuy` function allows you to create a new token and immediately execute buy orders:

```typescript
async createAndBuy(
  creator: Keypair,
  mint: Keypair,
  buyers: Keypair[],
  createTokenMetadata: CreateTokenMetadata,
  buyAmountSol: bigint,
  slippageBasisPoints: bigint = 300n,
  priorityFees?: PriorityFee,
  commitment: Commitment = DEFAULT_COMMITMENT,
  finality: Finality = DEFAULT_FINALITY,
): Promise<{ confirmed: boolean; jitoTxsignature?: string }>
```

This function performs the following steps:
1. Creates token metadata
2. Generates create instructions
3. Builds buy transactions for each buyer
4. Executes the create transaction
5. Submits all transactions to Jito for MEV protection

### Buying Tokens

To buy tokens, use the `buy` function:

```typescript
async buy(
  buyer: Keypair,
  mint: PublicKey,
  buyAmountSol: bigint,
  slippageBasisPoints: bigint = 500n,
  priorityFees?: PriorityFee,
  commitment: Commitment = DEFAULT_COMMITMENT,
  finality: Finality = DEFAULT_FINALITY,
): Promise<TransactionResult>
```

This function:
1. Generates buy instructions based on the SOL amount
2. Sends the transaction with optional priority fees
3. Returns the transaction result

### Selling Tokens

To sell tokens, use the `sell` function:

```typescript
async sell(
  seller: Keypair,
  mint: PublicKey,
  sellTokenAmount: bigint,
  slippageBasisPoints: bigint = 500n,
  priorityFees?: PriorityFee,
  commitment: Commitment = DEFAULT_COMMITMENT,
  finality: Finality = DEFAULT_FINALITY,
): Promise<TransactionResult>
```

This function:
1. Generates sell instructions based on the token amount
2. Sends the transaction with optional priority fees
3. Returns the transaction result

## Event Handling

The SDK provides an event system to keep you updated on various protocol activities:

```typescript
addEventListener<T extends PumpFundlerEventType>(
  eventType: T,
  callback: (
    event: PumpFundlerEventHandlers[T],
    slot: number,
    signature: string,
  ) => void,
): number
```

You can listen for the following event types:
- `createEvent`: Emitted when a new token is created
- `tradeEvent`: Emitted when a buy or sell trade occurs
- `completeEvent`: Emitted when a token's bonding curve is completed
- `setParamsEvent`: Emitted when global parameters are updated

To remove an event listener:

```typescript
removeEventListener(eventId: number): void
```

## Advanced Usage

### Creating Token Metadata

The SDK provides a method to create token metadata:

```typescript
async createTokenMetadata(create: CreateTokenMetadata): Promise<TokenMetadata>
```

This function uploads the token's metadata to IPFS and returns the metadata URI.

### Fetching Account Information

You can retrieve information about bonding curve and global accounts:

```typescript
async getBondingCurveAccount(
  mint: PublicKey,
  commitment: Commitment = DEFAULT_COMMITMENT,
): Promise<BondingCurveAccount | null>

async getGlobalAccount(
  commitment: Commitment = DEFAULT_COMMITMENT,
): Promise<GlobalAccount>
```

These methods allow you to access detailed information about the state of the PumpFun protocol and specific tokens.
